# Project Structure

The Evaluation Package is strictly organized to showcase the technical pipeline of the KRISHi-EYE project running on the Raspberry Pi 5 Edge NPU, abstracting away web-application development components.

## Directory Breakdown

### `01_overview/`
Placeholder for any immediate executive summaries, though primary overviews are located within `08_docs/README.md`.

### `02_deployment/`
The executable orchestration logic that runs live inference on the field.
- **`main_edge_inference_pipeline.py`**: The definitive live inference script combining Leaf Segmentation, Disease Classification, Lesion UNet masking, and Depth integration into a unified pipeline controlling UART outputs.
- **`deployment_orchestrator.py`**: The base orchestration blueprint for managing crop detection streams.
- **`depth_test_hailo*.py`**: Testing suites focused specifically on stereo calibration integration and SCDepthV3 mapping into real-world metric scales (Meters).
- **`validate_hailo_models.py`**: Validation scripts ensuring the NPU properly loads individual HEF models independently.
- **`camera_diagnostic.py`**: GStreamer / V4L2 backend validator resolving native RPi 5 camera indices and resolution compatibility.

### `03_training/`
Scripts necessary for scaling precision models down to edge deployment contexts.
- **`export_calibration_yolo.py` & `export_calibration_classifier.py`**: Select random subsets of training images to act as unsigned `uint8` [0-255] calibration representations. These enable INT8 quantized NPU compilation inside the Hailo DFC.

### `04_models/`
- **`onnx/`**: FP32 reference models natively structured with fixed static shapes and an ONNX v11 opset to guarantee compilation support.
- **`hef/`**: The ultimate native binaries loaded onto the accelerator (INT8 quantized versions optimized for real-time throughput on the Hailo-8L core). Include:
  - `lesion_segmentation_unet.hef`
  - `disease_classifier_mobilenet.hef`
  - `leaf_segmentation_yolov8n.hef`

### `05_configs/`
Core requirements and definitions framing the logic state.
- **`edge_requirements.txt`**: Standard Python environment requirements for running edge logic.
- **`model_capabilities.yaml`**: Internal reference tracking the reasoning properties spanning across potential pipeline states.

### `06_scripts/`
Supplementary tools utilized during the continuous development to test outputs without spinning up the heavy deployment suite. Include bash shells automating model conversions (`compile_all_to_hef.sh`) across the DFC API.

### `07_hardware_calibration/`
Outputs and external sub-components reflecting integration performance.
- Profiles measuring INT8 allocation and execution optimization generated by the compiler layers.
- **`krishi_eye_teensy.ino`**: The C++ Arduino firmware written for the Teensy 4.1. This code handles the inbound UART buffer mapping packet strings back to precise stepper servo actuations aligning the liquid pump nozzle mechanism towards target crop anomalies.

### `08_docs/`
You are here. The complete compendium framing evaluation material outlining architectural intent, data flow patterns, technical environment realities, and file manifests.
